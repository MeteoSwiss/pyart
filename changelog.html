Changelog#

v1.9.2#

  • FIX: several fixes for numpy 2 by @wolfidan in https://github.com/MeteoSwiss/pyart/pull/43

  • Dev by @wolfidan in https://github.com/MeteoSwiss/pyart/pull/44

Full Changelog: https://github.com/MeteoSwiss/pyart/compare/v1.9.0…v1.9.2

v1.9.0#

Improvements

  • modification to odim reader writer to flip coordinates instead of data

Bug fixes

  • coerce bool to int in cfradial writer

v1.8.9#

Improvements

  • improvement in dealias_spectra to give priority to neg velocities

  • updated output_to_geotiff.py to match ARM-DOE

  • various improvemnets to odim reader/writer to make it consistent on read/write + unit test

v1.8.8#

Bug fixes:

  • added metranet lib download in gh actions

  • added bytes-info when reading with metranet c-reader

New additions

  • add unit tests for metranet readers + possibility to read KE from cfg file

v1.8.7#

Bug fixes:

  • fixed python metranet reader to handle use of more than one byte in encoding

  • accept ‘c’ argument instead of ‘C’ for metranet reader

  • fixed typo in odim_h5_writer for sw_version

  • fix bug in dealias_spectra that led to only the first timestep being processed

  • Fixed issue in dealias_spectra with wrong output dtype

  • fix deprecation warning for scipy cumtrapz

New additions

  • add info on celsius for temp field in hydro class

  • added reader for Meteoswiss VAD products

  • [add gecsx fields to default_config.py](https://github.com/MeteoSwiss/pyart/commit/57664ce7b27c27af03f631fcd0e92a2

v1.8.6#

Bug fixes:

  • fixed argument warning in np.linalg.lstsq in wind.py

  • fix matplotlib deprecation warning in pyart/config.py

  • fixed error in bias_and_noise.py in case of missing radar.instrument_parameters

  • fixed error in attenuation.py in case of missing radar.instrument_parameters

v1.8.5#

Bug fixes:

  • fixes in gecsx for irregular azimuth angles in vol scan

  • correct reorder of data in odim

  • fix bug in time reading of metranet file

  • bug corrections in skyecho.py. Added function extract_sweeps_skyecho in skyecho.py

New additions

  • added file pyart/aux_io/knmi_h5.py with a reader of the KNMI H5 gridded radar data

  • added reader for SkyEcho proprietary netcdf data

  • add reader for SwissBirdRadar spectral data

  • add dealias routine for Doppler spectrum

v1.8.4#

Bug fixes:

  • fix in odim h5 reader and writer

  • correct reorder of data in odim

New additions

  • added windshear processing for lidar data

  • changed latlon plots to cross_section in GridMapDisplay, to be consistent with Py-ART ARM

v1.8.3#

Bug fixes:

  • fix in _rsl_interface.pyx for Cython 3.0

New additions:

  • added support for older python versions in pyart/testing

v1.8.2#

Bug fixes:

  • update of metranet read_product code for new cartesian format

  • fixes for change in cython 3.0 behaviour

  • Various deprecation fixes (matplotlib, numpy)

New additions

  • added conv_strat_yuter function from ARM Py-ART

  • Merge pull request https://github.com/MeteoSwiss/pyart/pull/21 from juhi24/dataset-patch

v1.8.1#

Bug fixes:

  • fix in radar_utils, replaced latitude by altitude as it was duplicated

  • fix for deprecated matplotlib register_cmap

  • fix of a bug in gecsx that lead to always using raster oversampling

  • fix scipy deprecation warning in ml.py

New additions

  • added option coerce_angles in join_radars to account for antenna misposition

v1.8#

Bug fixes:

  • improvement in colobar label of pyart/graph/gridmapdisplay.py

  • added _label_axes_latlon in class GridMapDisplay. Minor changes to improve grid plots

  • update rad4alp_gif_reader.py for new imageio version

  • Fix in gecsx oversampling, nrows and ncols were flipped

New additions:

  • improvements to vad.py, option for sign + multi-sweep Browning VAD

  • changed rad4alp gif lookup to make it agree with one of CPC

v1.7.1#

Bug fixes:

  • bug correction when masking undetect and nodata in odim_h5 reader

v1.7#

New additions:

  • New plot function: plot_xsection in RadarDisplay which can be used to display a cross-section of polar data between arbitrary coordinates

v1.6.4#

New additions#

New additions:

  • Support for new RainForest ODIM variable name

v1.6.3#

Bug fixes:

  • bug correction in compute_refl_time_avg in vpr.py: it was crashing when the elevation to average was not in the current scan

  • bug correction in vpr.py compute_refl_time_avg we set the radar_ou.time[‘data’] to np.zeros(nrays) to account for elevations to average not in the current radar

New additions:

  • VPR retrieval: better care of special case when melting layer peak has a value of 1

  • support for multi field Grid in odim h5 grid writer

  • change of datatype for time from float32 to float in read_odim_h5

  • better support of OPERA guidelines in odim writer and reader

  • Added possibility to specify ODIM convention in ODIM writers

v1.6.2#

Bug fixes:

  • Fixed an issue that was prohibiting ODIM grid files written by pyart to be read by pyart again

v1.6.1#

Bug fixes

  • fixed a bug in write_odim_grid that was using a time format different from the other pyart readers/writers

v1.6#

Bug fixes:

  • adaptation of the Cython codes of the RSL interface and 4DD to make them run with setuptools

  • rounding of the angles to a precision of 6 digits in the interpol_field code so we avoid floating point issues in the interpolation range (leads to northernmost ray being empty in some cases)

New additions:

  • new functions to accumulated gridded data and get data at multiple points in a grid

  • addition of a new spatialized VPR function
